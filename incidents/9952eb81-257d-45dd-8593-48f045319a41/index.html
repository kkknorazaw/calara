Root Cause Analysis

CyrusTreasury Exit Drain

1. Incident Overview TL;DR

Cyrus Treasury paid a matured Cyrus position from shared Pancake V3 liquidity using a spot-price-derived liquidity calculation that can be manipulated inside the same transaction. In exploit tx 0x2b7efdac5f052ee9a8f6de8f966b948027b76f7cc183e4868c98c7afc2d69524, the attacker-controlled helper exited CyrusPosition #15505 for a nominal 1707.055 USDT payout, but Treasury actually released 1289.353699758995692966 ETH plus 1707.055 USDT from Treasury-managed liquidity. The root cause is that withdrawUSDTFromAny sizes liquidity from flash-manipulable slot0() state and only meters the USDT leg, while collect() transfers both legs to the caller.

2. Key Background

Cyrus uses transferable ERC721 CyrusPosition NFTs to represent user positions, while CyrusTreasury manages a shared set of Pancake V3 LP NFTs for payout liquidity. Treasury authorization for exit, claimRewards, and compound depends on the current ownerOf(tokenId), so a valid holder or approved operator of a matured position can trigger exit. The exploited position was CyrusPosition #15505, which had strategyId = 0, , , and , meaning the position was mature by the exploit block.

3. Vulnerability Analysis & Root Cause Summary

The vulnerability is an accounting failure in Treasury’s USDT-denominated exit path. exit() computes the nominal amount owed to the matured position and then calls withdrawUSDTFromAny(toWithdraw, msg.sender). Inside that function, Treasury reads the live Pancake V3 pool price from slot0(), derives a notional availableUSDT, and scales LP liquidity from that spot-priced amount. Because the price is read inside the attacker-controlled transaction, the attacker can flash-manipulate it. Treasury then calls decreaseLiquidity with a minimum only on the USDT side and collect() sends both assets to the caller. Only the USDT amount is accumulated into totalWithdrawn, so the paired ETH leg is effectively unbounded profit.

function exit(uint256 tokenId) external nonReentrant {
    require(CyrusPositionManager.ownerOf(tokenId) == address(msg.sender));
    (uint256 totalAmount, PositionInfo memory positionInfo) = getPendingRewards(tokenId);
    uint256 toWithdraw = totalAmount + positionInfo.amount - feeAmount;
    withdrawUSDTFromAny(toWithdraw, msg.sender);
}

function withdrawUSDTFromAny(uint256 usdtAmountWithSlippage, address to) internal {
    (uint160 sqrtPriceX96,,,,,,) = IPancakePool(pool).slot0();
    (uint256 amount0, uint256 amount1) = PancakeSwapUtil.getAmountsForLiquidity(
        sqrtPriceX96, sqrtRatioAX96, sqrtRatioBX96, liquidity
    );
    uint256 availableUSDT = isToken0USDT ? amount0 : amount1;
    liquidityToUse = uint128((uint256(liquidity) * remaining) / availableUSDT);
    usdtReceived = decreaseLiquidity(tokenId, liquidityToUse, 0, minAmount, to);
    totalWithdrawn += usdtReceived;
}

4. Detailed Root Cause Analysis

The critical safety invariant is straightforward: exiting a position for toWithdraw USDT must not release more Treasury-managed asset value than that position is entitled to, and the sizing step must remain safe against same-transaction price manipulation. CyrusTreasury violates that invariant because it treats a manipulated spot quote as if it were a safe accounting oracle.

The verified source shows the exact breakpoint. withdrawUSDTFromAny iterates over Treasury LP NFTs, obtains the pool address, reads slot0(), computes amount0/amount1 for the full liquidity position, and uses the USDT leg alone to decide how much liquidity to burn. decreaseLiquidity then forwards both collected assets to the recipient:

(uint256 amount00, uint256 amount11) = PancakePositionManager.collect(
    CollectParams({
        tokenId: tokenId,
        recipient: to,
        amount0Max: type(uint128).max,
        amount1Max: type(uint128).max
    })
);

usdtWithdrawn = isToken0USDT ? amount00 : amount11;

The exploit trace shows this mismatch materializing exactly as the code predicts. Treasury considered the exit worth only 1707055000000000000000 units of USDT, but the manipulated LP burn released both tokens:

PancakeV3Pool::collect(..., 1289353699758995692966, 1707055000000000000000)
emit Transfer(... to: 0x938DBbB69e71D00f52d5eD5D69Ba892Fa1448a7B, value: 1289353699758995692966)
emit Transfer(... to: 0x938DBbB69e71D00f52d5eD5D69Ba892Fa1448a7B, value: 1707055000000000000000)
emit Exited(..., 1707055000000000000000, ..., 15505)

That trace is the determinative evidence: the protocol’s accounting event says 1707.055 USDT, but Treasury actually sends out a second, far larger ETH leg that is never charged against the exit budget.

5. Adversary Flow Analysis

The adversary sequence had three observable stages. First, the victim holder approved the attacker helper on CyrusPosition. Second, tx 0x9395d7522aec224137eb39915f67dafab044d5400910bbc9a8ce2de9e27c5992 created helper 0x938dbbb69e71d00f52d5ed5d69ba892fa1448a7b, which the collector trace summary identifies as the helper creation transaction. Third, exploit tx 0x2b7efdac5f052ee9a8f6de8f966b948027b76f7cc183e4868c98c7afc2d69524 performed the flash manipulation and exit.

In the exploit tx, the helper borrowed 1798 ETH from Pancake V3, swapped it into USDT to distort the ETH/USDT price, exited CyrusPosition #15505, received 1289.353699758995692966 ETH and 1707.055 USDT from Treasury-managed liquidity, swapped 760000 USDT back into ETH, repaid 1799.0788 ETH to the flash pool, and forwarded the residual assets to wallet 0xf96eb14171b71ac16200013753dff3e91043b63b.

This is an ACT opportunity because the attacker used only public state, a public victim approval, public AMM liquidity, and ordinary transaction capabilities. No privileged keys or off-chain secrets were required.
