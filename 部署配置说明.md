# 部署配置说明

当前目录已经补齐以下可直接使用的部署配置：

- `vercel.json`：Vercel 直接导入项目可用
- `netlify.toml`：Netlify 直接导入项目可用
- `nginx.conf`：Linux + Nginx 自建服务器可用
- `Caddyfile`：Linux + Caddy 自建服务器可用
- `web.config`：Windows + IIS 可用

## 你只需要做一件事

按你的部署环境，使用对应的那一个配置文件即可，不需要全部使用。

## 各环境最短发布方式

1. Vercel
- 把整个目录上传到仓库
- 在 Vercel 导入仓库
- 框架类型选 `Other`
- 输出目录保持默认（仓库根目录）

2. Netlify
- 把整个目录上传到仓库
- 在 Netlify 导入仓库
- 构建命令留空
- Publish directory 填 `.`

3. Nginx
- 把站点文件放到 `root` 对应目录（示例是 `/var/www/wan`）
- 按实际域名修改 `server_name`
- 重载 Nginx

4. Caddy
- 把站点文件放到 `root` 对应目录（示例是 `/var/www/wan`）
- 按实际域名替换 `:80` 为你的域名
- 重载 Caddy

5. IIS
- 站点根目录指向当前目录
- 确保 URL Rewrite 模块已安装
- `web.config` 自动生效

## 说明

- `api/stripe/*` 与登录相关后端能力已做静态占位页，静态托管不会 404，但不会有真实后端功能。
- 页面浏览、静态内容访问、已有事故详情路由可直接使用。
